## 使用限制

SQLPub 提供 Serverless MySQL 数据库服务，同时设定了一些资源和操作限制以保证平台稳定性和性能。

### 核心限制

- **数据库数量**：每个账号最多可创建 10 个数据库实例（免费计划可能更少）。
- **并发连接数**：
  - 免费计划：最多 50 个连接
  - 标准/付费计划：根据资源配置动态调整
- **存储上限**：
  - 免费计划：最大 5GB
  - 标准计划：最大 100GB
- **单表行数**：单表行数建议不超过 10,000,000 行
- **查询限制**：
  - 每秒最大查询数：根据实例规格不同，通常为 1000 QPS
  - 大型查询建议使用分页或批量处理

### 资源限制与伸缩

- Serverless 模式支持自动扩缩容，但仍受上述最大值限制。
- 超过限制的操作可能会被拒绝或排队处理。

### API 调用限制

- 每个账号每分钟最大请求次数：
  - 免费计划：100 次/分钟
  - 付费计划：1000 次/分钟
- 建议对高频调用进行节流处理，避免触发限流。

### 注意事项

- 请根据实际业务需求合理创建数据库和表。
- 及时监控数据库负载和存储使用情况。
- 对于高频或大型操作，请使用批量或分页处理方式以优化性能。

### 小结

了解并遵守 SQLPub 的使用限制，可以确保数据库的稳定性和性能，同时避免触发资源和 API 限制。
